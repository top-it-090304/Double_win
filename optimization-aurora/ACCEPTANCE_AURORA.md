# Приёмка перед релизом на ОС Аврора (Honor of Aurora)

Документ для владельца проекта и тестировщика: проверка, что режим **«На тапке»** и связанные оптимизации работают согласованно, сохранения не ломаются, нет критичных регрессий.

**Связанные артефакты:** [TASKS_INDEX.md](TASKS_INDEX.md), [JOURNAL.md](JOURNAL.md) — на дату последнего ревью задач по «волне» оптимизации все задачи TASK-001…TASK-009 в журнале имеют статус **выполнено**; отменённых задач нет.

---

## Чек-лист (ручной прогон)

Отметьте **Pass** или **Fail** в последней колонке. При **Fail** кратко опишите шаг и устройство/сборку.

| # | Проверка | Ожидаемый результат | Pass / Fail |
|---|----------|----------------------|-------------|
| 1 | **Первый запуск / миграция сохранения** | Игра стартует без ошибок; при отсутствии сохранения создаётся профиль по умолчанию; при наличии старого сохранения с `performance_mode` 0..3 значения не «ломаются» (режимы «Минимальный / Средний / Максимальный» ведут себя как раньше). При необходимости сменить режим, выйти в ОС, снова запустить — настройка производительности сохранена. | |
| 2 | **Переключение пресетов производительности** | В **Настройки** последовательно выбрать: Минимальный → Средний → Максимальный → **На тапке** → снова Минимальный. При каждом выборе применяются ожидаемые изменения (FPS/подсказка в UI, без зависаний); для **«На тапке»** видна подсказка про режим для слабого железа (по журналу: 30 FPS, физика 30, Y-sort реже, viewport stretch и т.д.). | |
| 3 | **«На тапке» после перезапуска** | Выставить **На тапке**, сохранить настройки (если есть явное сохранение), полностью закрыть игру, открыть снова — режим **остаётся «На тапке»**. | |
| 4 | **Остров с дождём: логика** | В условиях активной погоды «дождь» (когда по правилам игры действует множитель награды за убийство монстров) убедиться, что **механика награды** не сломана: при дожде бонус к луту по-прежнему применяется (см. TASK-003 в журнале). | |
| 5 | **Остров с дождём: визуал** | На пресетах **не** «На тапке» при дожде виден оверлей дождя (частицы/эффект), без розовых текстур и вылетов. В режиме **«На тапке»** оверлей дождя **отключён** (меньше нагрузки), при этом п. 4 по логике награды должен оставаться корректным. | |
| 6 | **Бой с толпой юнитов (FPS)** | На «На тапке» и в сравнении с «Минимальный» (по желанию) провести бой с большим числом юнитов на экране: кадр остаётся играбельным, нет длительных фризов; при падении FPS зафиксировать сцену и устройство. | |
| 7 | **Главное меню** | Переходы в меню, настройки, старт/загрузка игры — без чёрного экрана, без зависаний UI. | |
| 8 | **HUD в игре** | Верхняя полоса (ресурсы, HP и т.д.) обновляется корректно; на мобильном управлении тач-оверлей появляется/скрывается при смене локации без «залипания» видимости. | |
| 9 | **Звук** | В бою и в меню: музыка и SFX без постоянных обрывков/треска; в режиме «На тапке» дополнительно допустимо **реже** звучание второстепенных эффектов (политика по журналу TASK-006), но критичные сигналы (урон игроку, смерть, UI) должны оставаться слышимыми. | |
| 10 | **Выход и повторный вход** | Выйти из игры штатно (меню выхода или аналог), снова запустить — настройки, прогресс, режим производительности не «сбрасываются» неожиданно. | |

## Известные ограничения (не обязательно дефекты)

- **Y-sort в режиме «На тапке»:** интервал обновления глобальной Y-сортировки увеличен (например, раз в **8** кадров против меньшего интервала у других пресетов). В редких случаях возможен **кратковременный артефакт порядка слоёв** (объекты «перескакивают» при движении). Рекомендация из журнала: при проблемах на целевом устройстве согласовать настройку интервала отдельно.
- **Дождь на «На тапке»:** визуальный оверлей отключён ради FPS; **логика награды при дожде** сохраняется — игрок может не видеть частицы, но правила лута остаются.
- **Внутренний масштаб рендера / viewport stretch** в «На тапке» снижает разрешение буфера — возможна **чуть более мягкая картинка**; при необходимости точечная подстройка на устройстве Авроры (см. журнал TASK-005).
- **Отсечение декора по расстоянию** (остров 1 и др.): в зоне границы радиуса теоретически возможно **кратковременное мелькание** скрытия/показа декора; при появлении — рассмотреть радиус/гистерезис (журнал TASK-008).
- **HUD / тач:** в «На тапке» обновление видимости тач-контролов **throttle** (не каждый кадр) — при очень быстрых сменах сцен возможна на доли секунды рассинхронизация до следующего цикла (см. журнал TASK-009).
