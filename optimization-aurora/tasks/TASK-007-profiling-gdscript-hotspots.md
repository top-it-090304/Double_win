# TASK-007 — Профилирование и снижение аллокаций в GDScript

## Цель

Найти **горячие** места в GDScript (каждый кадр `_process` / `_physics_process`), уменьшить создание массивов/словарей/string в кадре, особенно при режиме «На тапке» (или глобально, если безопасно).

## Метод

1. Использовать встроенный **Profiler** Godot 4: Functions / Script Time / Memory (краткая выжимка в журнале: какие функции топ).
2. Поиск по коду: `+=` строк, `.append` в циклах без пула, `get_nodes_in_group` каждый кадр, лишние `signal` аллокации.
3. Внести **точечные** правки в 1–3 самых дорогих местах с измеримым эффектом.

## Требования (MUST)

- Каждая правка: до/после **коротко** в журнале (что ожидалось).
- Не менять геймплейные константы (урон, скорость) — только производительность.
- Если оптимизация специфична для «На тапке» (например реже обновлять вспомогательный кэш) — обернуть в проверку пресета.

## Критерии приёмки

- Снижение времени в профайлере для затронутых функций **или** меньше аллокаций за кадр в Memory profiler.

## Ожидаемые зоны риска (подсказки, проверить фактом)

- `YSortManager` уже оптимизирует частоту; ищи другие глобальные циклы по дереву сцены.
- AI, спавнеры врагов, инвентарь/ресурсы на базе.
