# TASK-010 — Критерии приёмки и чек-лист перед релизом на ОС Аврора

## Цель

Сформировать **проверяемый** список для владельца проекта и тестировщика: все оптимизации режима **«На тапке»** работают вместе, сохранения совместимы, нет регрессий.

## Тип задачи

В основном **документирование и проверка**; правки коду — только если найдены блокеры.

## Требования (MUST)

1. Пройти по `optimization-aurora/JOURNAL.md` и `TASKS_INDEX.md`: все задачи со статусом «сделано» или явно «отменено» с причиной.
2. Составить **чек-лист** (markdown-таблица или список) внутри этого репозитория — предложить путь: дополнение `optimization-aurora/README.md` **или** новый файл `optimization-aurora/ACCEPTANCE_AURORA.md` (выбрать один, не дублировать).
3. Чек-лист должен включать минимум:
   - первый запуск / миграция сохранения;
   - переключение каждого пресета производительности, включая «На тапке»;
   - остров с дождём (логика + визуал);
   - бой с толпой юнитов (FPS);
   - главное меню и HUD;
   - звук (нет обрывов);
   - выход из игры и повторный вход.
4. Зафиксировать **известные ограничения** (например: «Y-sort раз в 8 кадров даёт редкий артефакт слоёв»).

## Критерии приёмки

- Владелец может пройти чек-лист без знания кода.
- Все пункты имеют ожидаемый результат (pass/fail).

## Выходной артефакт

- Один согласованный документ с чек-листом + ссылка на него из `README.md`.
