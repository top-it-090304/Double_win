# TASK-006 — Звук: бюджет и упрощения в режиме «На тапке»

## Цель

Снизить нагрузку на CPU/микширование и избежать **звуковых** просадок при множестве одновременных событий на слабом железе.

## Контекст

- `honor-of-aurora/autoload/SoundManager.gd` — центральное место для громкости и воспроизведения.
- Поиск: `AudioStreamPlayer`, `play()`, пулы, одновременные SFX.

## Требования (MUST)

1. Проанализировать `SoundManager` и типичные вызовы: есть ли **лимит** одновременных голосов, приоритеты, отсечение дальних звуков.
2. Для режима «На тапке» ввести **хотя бы одно**:
   - ограничение максимального числа одновременных `AudioStreamPlayer2D` / SFX;
   - приоритет: UI и диалоги громче/надёжнее, второстепенные ambient режутся;
   - пониженная частота дискретных «мелких» звуков (шаги, удары) через throttle/debounce **без** полной потери обратной связи.
3. Не ломать громкости из настроек (`volume_*` в SaveManager).

## Критерии приёмки

- При стресс-сцене (много боёв) меньше аллокаций/вызовов в профайлере звука **или** субъективно стабильнее кадр.
- Нет вечного «залипания» зацикленных стримов.

## Файлы (ожидаемо)

- `honor-of-aurora/autoload/SoundManager.gd`
- Ключевые игровые сцены, если звуки создаются локально без SoundManager
