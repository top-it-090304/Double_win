# TASK-005 — Масштаб рендера / viewport под режим «На тапке»

## Цель

Снизить число пикселей, которые GPU рисует каждый кадр, для пресета **«На тапке»**: внутреннее разрешение ниже нативного, с корректным UI.

## Контекст

- `project.godot` `[display]`: `viewport_width/height` 1280×720, `stretch/mode` canvas_items.
- `Save_manager.gd`: `ui_scale_percent`, `apply_window_and_engine_settings` — возможно уже есть масштаб окна/контента.

## Требования (MUST)

1. Найти **существующий** механизм масштабирования окна/UI (`content_scale_factor`, SubViewport, или CanvasLayer). Не изобретать второй параллельный путь без причины.
2. Для режима «На тапке» задать **сниженное** внутреннее разрешение рендера (например 0.75 или 0.5 от текущего) **или** эквивалент через `Window.content_scale_factor` / stretch — выбрать один согласованный подход с Godot 4.4 и GL Compatibility.
3. **UI** не должен стать нечитаемым: проверить главное меню и один игровой HUD.
4. Все переключения — только при `performance_mode` = «На тапке»; остальные пресеты без регрессий.

## Критерии приёмки

- Визуально: картинка чуть мягче/пиксельнее, но стабильный FPS на слабом устройстве (описать в журнале, если устройства нет — логическое обоснование).
- Нет чёрных полос или неверного hit-test кнопок.

## Файлы (ожидаемо)

- `honor-of-aurora/autoload/Save_manager.gd` — методы применения окна
- Возможно `honor-of-aurora/autoload/GameManager.gd` или корневая сцена
- `project.godot` — только если нужен дефолт, не ломая другие режимы
