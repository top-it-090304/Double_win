# TASK-001 — Режим «На тапке»: ядро (enum, сохранение, UI)

## Цель

Добавить в игру **новый** пресет производительности с отображаемым именем **«На тапке»**, чтобы игрок мог выбрать его в настройках и значение сохранялось вместе с остальными настройками.

## Контекст (факты из кодовой базы)

- `PerformancePreset.Mode` в `honor-of-aurora/autoload/performance_preset.gd`: сейчас `MINIMAL=0`, `MEDIUM=1`, `MAXIMUM=2`, `CUSTOM=3`. `clamp_mode` ограничивает диапазон; `CUSTOM` в сохранениях мапится в средний в `Save_manager.gd`.
- `SaveManager.performance_mode` сериализуется в `SAVE_DATA`.
- UI: `honor-of-aurora/ui/menu/settings_overlay.gd` — метод `_fill_performance_options()` добавляет три строки; `_update_performance_desc()` — описания для режимов.

## Требования (MUST)

1. Ввести **новый** режим в enum (имя константы на английском, например `SLIPPER` или `AURORA_TAPKI` — выбери одно и используй везде). Отображаемое имя в UI: **строго** «На тапке».
2. Обновить `clamp_mode` и любые `clamp(..., 0, 2)` в настройках так, чтобы новый индекс был валиден.
3. В `settings_overlay`: добавить пункт в `OptionButton`, расширить `_update_performance_desc()` — **краткое** описание для пользователя (1–2 предложения: максимальная экономия ресурсов, возможны упрощения графики и эффектов).
4. **Миграция:** старые сохранения без нового значения не должны ломаться. Явно описать в комментарии в коде: что происходит со старыми `performance_mode` 0..3.
5. В `PerformancePreset.apply_from_save_manager` пока можно задать **временные** те же числа, что у `MINIMAL`, **или** явный TODO с комментарием «TASK-002 настроит параметры» — но режим должен **различаться** по `mode` в `match`, чтобы следующие задачи могли вешать условия `mode == Mode.SLIPPER`.

## Не делать в этой задаче

- Финальные значения FPS/физики/Y-sort (это TASK-002).
- Отключение дождя и частиц (TASK-003).

## Критерии приёмки

- В редакторе Godot: настройки показывают «На тапке», выбор сохраняется после перезапуска (через существующий `save_game`).
- Нет ошибок при загрузке старого сохранения.

## Файлы для правки (ожидаемо)

- `honor-of-aurora/autoload/performance_preset.gd`
- `honor-of-aurora/autoload/Save_manager.gd` (если нужны комментарии миграции / default)
- `honor-of-aurora/ui/menu/settings_overlay.gd`
- При необходимости: `honor-of-aurora/ui/menu/settings_overlay.tscn` (если жёстко задано число элементов — проверить)
