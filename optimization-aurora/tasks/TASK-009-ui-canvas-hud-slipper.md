# TASK-009 — UI: CanvasLayer, HUD, лишние перерисовки в режиме «На тапке»

## Цель

Снизить стоимость UI-слоя на слабом GPU: реже обновлять текстуры, уменьшить количество полноэкранных слоёв, отключить тяжёлые эффекты на HUD.

## Контекст

- `honor-of-aurora/ui/HUD/` и связанные сцены.
- Уже есть паттерн слоёв в меню (`settings_overlay`, CanvasLayer).

## Требования (MUST)

1. Инвентаризировать HUD: что обновляется каждый кадр (`_process` с постоянным `text =` или `queue_redraw`).
2. Для «На тапке»: обновлять реже **не критичные** элементы (таймеры, второстепенные индикаторы) — throttle 2–5 кадров.
3. Отключить или упростить **шейдеры/тени** на UI, если есть, только в этом пресете.

## Критерии приёмки

- HUD остаётся читаемым; критичные значения (HP, ресурсы) обновляются достаточно часто.
- Меньше вызовов `CanvasItem` redraw в профайлере при типичном геймплее.

## Файлы (ожидаемо)

- Скрипты под `honor-of-aurora/ui/HUD/`
- При необходимости базовые классы UI
