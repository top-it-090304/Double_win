# TASK-004 — Импорт текстур: сжатие, mipmaps, лимиты для слабого GPU

## Цель

Уменьшить потребление **VRAM** и стоимость сэмплинга на цели «Аврора», не ломая визуал критичных объектов.

## Контекст

- В Godot 4 параметры импорта хранятся в файлах `*.import` рядом с текстурами.
- В проекте уже встречается `compress/mode`, `mipmaps/generate`, `process/size_limit`, `vram_compression` в `project.godot`.

## Требования (MUST)

1. **Не** массово переписывать тысячи файлов без стратегии: сначала выбрать **категории** (например: фоновые тайлы, декор, UI-крупные, персонажи).
2. Для выбранной категории применить:
   - включение **VRAM-сжатия**, совместимого с GL Compatibility и мобильными GPU (ориентир — существующие настройки проекта);
   - по возможности **mipmaps** для уменьшаемых вдали спрайтов (с осторожностью для пиксель-арта — не размывать критичный стиль);
   - `process/size_limit` для очень больших текстур, где полное разрешение не видно на экране телефона.
3. Документировать в журнале: **какие папки** затронуты и **почему**.

## Ограничения

- Сохранить читаемость ключевого UI и иконок.
- После изменений прогнать открытие хотя бы **одной** игровой сцены и меню — нет розовых/битых текстур.

## Критерии приёмки

- Уменьшенный размер импортированных ресурсов или VRAM по профайлеру (если доступен) — либо обоснованное «ожидаемо меньше» по `size_limit`/сжатию.

## Метод работы

- Предпочтительно: скрипт/инструкция для **группового** обновления `.import` в выбранных папках **или** настройки дефолта для новых импортов — что уместнее в Godot 4.4 для этого репозитория.
