# Оптимизация под ОС «Аврора» (Honor of Aurora)

Центральная папка процесса: задачи, общий контекст, журнал прогресса.

## Как пользоваться

| Файл | Назначение |
|------|------------|
| [CONTEXT.md](CONTEXT.md) | Полный контекст проекта и целей — прочитать перед работой или приложить к обсуждению |
| [PROMPT_FOR_NEW_CHAT.md](PROMPT_FOR_NEW_CHAT.md) | **Готовый промпт для каждой задачи** (TASK-001…010) — копировать один блок в начало сессии |
| [TASKS_INDEX.md](TASKS_INDEX.md) | Реестр задач, статусы, ссылки на ТЗ |
| [tasks/](tasks/) | Отдельное ТЗ на каждую задачу |
| [JOURNAL.md](JOURNAL.md) | Журнал: что сделано, какие файлы |
| [ACCEPTANCE_AURORA.md](ACCEPTANCE_AURORA.md) | **Чек-лист приёмки** перед релизом на ОС Аврора (ручной прогон, известные ограничения) |

## Цель продукта

Режим производительности **«На тапке»** в настройках игры: при включении активируются все согласованные оптимизации для очень слабого железа (порт на ОС Аврора).

Код игры: каталог `honor-of-aurora/` (Godot 4.4, GL Compatibility).

## Порядок работы

1. Выбрать задачу в `TASKS_INDEX.md` со статусом «не начато» или «в работе».
2. Открыть новую сессию, скопировать **готовый блок** для этой задачи из `PROMPT_FOR_NEW_CHAT.md` (раздел TASK-001 … TASK-010).
3. Иметь под рукой `optimization-aurora/CONTEXT.md` и соответствующий `optimization-aurora/tasks/TASK-…-….md` из скопированного промпта.
4. После выполнения — внести запись в `JOURNAL.md` и обновить статус в `TASKS_INDEX.md`.
