# Рабочий на базе

## Текущая схема

На **`Game_base_islad`** при старте вызывается **`_setup_base_navigation_from_tile_collisions`**: выпечка **`NavigationPolygon`** из **статических коллизий** узла `island` (`NavigationServer2D.parse_source_geometry_data` + `bake_from_source_geometry_data`). Ручные дыры в пресете не нужны.

Рабочий по умолчанию идёт по **`map_get_path`** (`base_worker_use_physics_steering = false`). Режим только с лучами — запасной, если сетка не выпеклась.

---

## Простыми словами: что такое «регион»

- **Коллизия** (стены на карте) — это физика: персонаж **не может пройти сквозь** камень.
- **Навигация** — это **отдельная «карта ходьбы»** для ИИ: движок строит маршрут только по **голубой сетке** (где «разрешено идти»).

Узел **`NavigationRegion2D`** в Godot как раз задаёт эту карту: внутри него лежит **`NavigationPolygon`** — многоугольники «куда можно ходить». Без этого полигона (или если он один большой прямоугольник без дыр под стены) путь **режет стены** — потому что для навигации «там проход», а для глаз там стена.

**Регион навигации** = кусок сцены, который говорит серверу навигации: «вот эта область — проходимая сетка».

## Что значит «регион создаётся скриптом»

В `game_base_islad.gd` есть строка `IslandEncounterShared.attach_navigation_region(self)`.

Она делает так: **если** в сцене базы **ещё нет** узла с именем `IslandNavigationRegion`, движок **сам добавляет** его из файла `island_navigation_region.tscn`.

Сейчас узел **уже лежит в сцене** `Game_base_islad.tscn` — скрипт видит его и **не создаёт второй**. Ты правишь навигацию **в этом узле** в редакторе.

## Что уже сделано в проекте

- В корень **`Game_base_islad.tscn`** снова добавлен узел **`IslandNavigationRegion`** (сцена `island_navigation_region.tscn`).
- У него позиция **`(-592, 867)`** — сетка пресета совпадает с островом только если регион стоит так же (подвинь, если у тебя карта сдвинута).
- Включено **`use_base_island_preset = true`** — при запуске подставляется контур из **`base_island_nav_preset.gd`** (дыры под стены).

## Если рабочий всё равно упирается в стену

Сделай по порядку:

1. **Открой** `Game/Game_base_islad.tscn`, выдели **`IslandNavigationRegion`**, включи внизу **«Видимая навигация»** (Debug → Visible Navigation), **запусти сцену** и посмотри: **голубая сетка совпадает с землёй** и **обходит стены**? Если сетка «уехала» — **подвинь узел** `IslandNavigationRegion` (position), пока сетка не ляжет на остров.

2. Если пресет **устарел** относительно твоей карты — в инспекторе у **`IslandNavigationRegion`** сними галочку **`use_base_island_preset`**, открой **Navigation Polygon**, нарисуй **внешний контур** суши и **внутренние контуры** (дыры) по стенам, нажми **Bake NavigationPolygon**, сохрани сцену.

3. В **Navigation Polygon → Agent Radius** поставь **16–20**, снова **Bake** — персонаж визуально не должен «лезть» в стену по краю сетки.

4. Убедись, что файл **`base_island_nav_preset.gd`** есть в проекте (и в git), иначе пресет не загрузится.

## Где лежит код движения рабочего

`ally/pawn/scripts/pawn_base.gd` — путь считается через **`NavigationServer2D.map_get_path`**, слои как у **`NavigationAgent2D`** на пешке.

## Ошибка `merge a navigation mesh polygon edge` / `merge_rasterizer_cell_scale`

Обычно это **плохая геометрия** (пересекающиеся рёбра, дыры накладываются на контур) или **разный `cell_size`** у полигона и карты навигации. В проекте по умолчанию:

- **`use_preset_holes = false`** на `IslandNavigationRegion` — пресет **только внешний контур**, без дыр (дыры можно снова включить в инспекторе, когда отрисуешь их аккуратно в редакторе).
- После назначения полигона вызывается **`NavigationServer2D.map_set_cell_size`** под **`cell_size`** ресурса.
- У пешки **`base_move_use_map_force_update = false`** — принудительный sync карты отключён (его можно включить, если без него путь «отстаёт»).
- **Запасной режим:** на инстансе пешки включи **`base_move_use_navigation_agent_path`** — движение через **`get_next_path_position`**, без `map_get_path` (как старый вариант).

В **Project Settings** уже стоит `navigation/3d/merge_rasterizer_cell_scale = 0.001` (рекомендация из текста ошибки).

## Не отключай регион

У узла **`IslandNavigationRegion`** должно быть **`enabled = true`**, иначе навигации нет.
