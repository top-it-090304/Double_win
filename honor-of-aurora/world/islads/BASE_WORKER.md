# Рабочий (pawn) на базовом острове

## Контекст

Рабочий **ходит и добывает ресурсы только на базе** (`Game/Game_base_islad.tscn`), а на походных островах (уровни 1–5) ведёт себя как **обычный спутник** (следование, бой), без автоматической добычи и без использования сценной навигации базы.

Имеет смысл держать **узел навигации именно на сцене базы**: он описывает, где рабочий может ходить на карте базы и как обходить стены. На других островах этот узел не нужен.

## Что уже сделано в коде

1. В **`game_base_islad.gd`** при `_ready()` вызывается `IslandEncounterShared.attach_navigation_region(self)`: в корень сцены базы добавляется **`IslandNavigationRegion`**, если узла с таким именем ещё нет.

2. Сцена **`world/islads/island_navigation_region.tscn`** — это `NavigationRegion2D` с полигоном по умолчанию (большой прямоугольник). Его можно **заменить или дополнить** своим контуром только для базы.

3. **`IslandWorkerTargets`** ищет цели по дереву сцены базы: слои `rocks` / `rock_island`, `trees`, овцы/маркеры мяса (см. ниже).

4. **`pawn_base.gd`**: добыча с `NavigationAgent2D` включается **только** при `Events.current_location == BASE`.

## Что сделать вам в редакторе

### 1. Полигон навигации на базе (важно для обхода стен)

**Вариант A — править общий ресурс сцены навигации**  
Откройте `world/islads/island_navigation_region.tscn`, выберите корневой узел, в инспекторе отредактируйте **Navigation Polygon**: внешний контур — проходимая суша базы, внутренние контуры — «дыры» (стены, вода).

**Вариант B — только база, отдельный инстанс**  
Откройте `Game/Game_base_islad.tscn`, в корень добавьте инстанс `island_navigation_region.tscn`, **имя узла** должно быть **`IslandNavigationRegion`**. Тогда скрипт базы не создаст второй регион. Настройте полигон здесь — так удобнее, если контур базы отличается от «дефолтного» прямоугольника в `.tscn` ресурса.

Рекомендация: если база — единственное место добычи, **вариант B** обычно нагляднее: всё для базы лежит в `Game_base_islad.tscn`.

### 2. Мясо на базе

Разместите **`objects/world/sheep_resource.tscn`** и/или пустые `Node2D` в группе **`island_meat_marker`** — иначе для задачи «мясо» не будет найдена точка.

### 3. Походные острова

Ничего не настраивайте для рабочего: навигация с `game_level_*.gd` по-прежнему может использоваться **другими системами** (встречи и т.д.), на поведение рабочего это не влияет.

## Кратко

| Где | Поведение рабочего |
|-----|-------------------|
| **База** | Добыча по работе + навигация по `IslandNavigationRegion` |
| **Острова 1–5** | Обычный спутник, без автодобычи |
