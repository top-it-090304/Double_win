# TASK-017 — SLIPPER: тяжёлые визуальные эффекты в сценах (не импорт текстур)

## Цель

Отключить или упростить **в игровых сценах** тяжёлые визуальные эффекты **только в «На тапке»**: частицы `CPUParticles2D`/`GPUParticles2D`, шейдеры на декоре, лишние `CanvasItem` эффекты. **Не** заниматься массовой перенастройкой `.png.import` и экспортом — это отдельная тема портирования.

## Контекст

- Дождь уже отключён визуально в SLIPPER (`rain_system.gd`).
- В проекте могут быть другие частицы в окружении, базе, UI.

## Требования (MUST)

1. Проверка режима: `PerformancePreset.is_slipper_mode(SaveManager)`.
2. Поиск по `honor-of-aurora/` на `CPUParticles2D`, `GPUParticles2D`, тяжёлые `ShaderMaterial` в игровом цикле.
3. Для найденных эффектов (не критичных для геймплея):
   - `emitting = false` / `visible = false` / `process_mode = DISABLED` в SLIPPER при входе в сцену или через автозагрузку-менеджер;
   - при выходе из SLIPPER — восстановить состояние.
4. **Не** отключать эффекты, связанные с **обязательной** обратной связью (критический урон, предупреждение о смерти) — только после явной проверки.

## Не делать

- Не менять глобальные настройки рендера в `project.godot` для всех пресетов.
- Не проводить волну правок только `.import` файлов.

## Критерии приёмки

- На «На тапке» меньше нагрузка в сценах с частицами; в других пресетах визуал прежний.
- Журнал волны 2 содержит список отключённых/упрощённых эффектов.
