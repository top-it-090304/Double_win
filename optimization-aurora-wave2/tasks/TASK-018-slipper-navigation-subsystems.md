# TASK-018 — SLIPPER: навигация и вспомогательные подсистемы

## Цель

Снизить стоимость **навигации и вспомогательных** расчётов (NavigationAgent2D, обходы графа, периодические пересчёты путей) **только в «На тапке»**, уменьшив частоту обновлений или отключив второстепенные запросы.

## Контекст

- Острова и база могут использовать `NavigationRegion2D` / `NavigationAgent2D` (поиск по `honor-of-aurora/`).
- `IslandEncounterShared.attach_navigation_region` встречается в уровнях.

## Требования (MUST)

1. Только в SLIPPER.
2. Для `NavigationAgent2D` (и аналогов):
   - увеличить интервал `target_desired_distance` / `path_update` поведения **если** API позволяет без поломки API Godot 4.4;
   - или вызывать `get_next_path_position()` / обновление цели реже через **троттлинг в скрипте** юнита на дистанции от игрока (связать с TASK-015 при пересечении).
3. Не отключать навигацию для врагов в **ближнем** бою с игроком.
4. Документировать в журнале: какие сцены/типы юнитов затронуты.

## Не делать

- Не ломать телепорты и квестовые триггеры зон.
- Не менять геометрию навигационных мешей.

## Критерии приёмки

- Враги не «застывают» в агрессивном радиусе от игрока.
- На слабом железе меньше просадок при толпе врагов в SLIPPER.
- Запись в `optimization-aurora-wave2/JOURNAL.md`.

## Файлы (ожидаемо)

- Скрипты врагов с навигацией, `honor-of-aurora/Game/` и `honor-of-aurora/enemies/`
