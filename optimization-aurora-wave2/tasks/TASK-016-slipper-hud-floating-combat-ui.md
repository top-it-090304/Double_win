# TASK-016 — SLIPPER: HUD и всплывающий урон без лишних обновлений

## Цель

Уменьшить стоимость отрисовки и обновления **интерфейса боя** в режиме **«На тапке»**: числа урона, полоски, иконки, которые могут спамить при большом числе юнитов.

## Контекст

- HUD: `honor-of-aurora/ui/HUD/`, группа `hud` (см. `Save_manager.apply_window_and_engine_settings`).
- Возможны всплывающие числа урона, плавающие лейблы, `queue_redraw` на Canvas.

## Требования (MUST)

1. Только в SLIPPER: `PerformancePreset.is_slipper_mode(SaveManager)`.
2. Ввести **лимит** одновременно отображаемых «всплывашек» урона (или аналогичных эффектов): при превышении — **не показывать** часть визуала, **без изменения** фактического урона (см. похожую идею в документации приёмки при наличии).
3. Реже обновлять **необязательные** элементы HUD (например второстепенные индикаторы), если обновление каждый кадр не нужно для читаемости.
4. Не ломать масштаб UI и `apply_user_ui_scale` / `content_scale_factor` логику.

## Не делать

- Не скрывать критичные индикаторы (HP, основные ресурсы) без согласования с дизайном — по умолчанию только **декоративный** перегруз и лимит чисел.

## Критерии приёмки

- При большом числе попаданий игра остаётся стабильнее по FPS; игрок всё ещё понимает, что происходит бой.
- Урон не визуализируется для части ударов — **ок**, если урон применён (проверить журналом и тестом).

## Файлы (ожидаемо)

- `honor-of-aurora/ui/HUD/` — поиск по `damage`, `float`, `Popup`, `Label`, `Particle`
